2. What is Angular.

  • It's a framework for building javascript / Typescript apps. Makes things easier, creates structure, easier to test

3. angular is a frontend app.

  • Generates html, css, Typescript, Angular. Data is not stored in angular, the data is store in the back end.
  • Use http to get to the api backend. Data and business logic is in the backend, front is for Presentation logic

4. What's needed to build an Angular

  • To build an Angular App you need NPM, to get install the latest version of NODE (nodejs.org). Min version 6.5 for node.
  • type: npm install -g @angular/cli : You may need to precede with sudo if on linux. To check the version type ng --version

5. Create your first app.

  • type: ng new yourprojectname
  • type: ng serve --host 10.27.10.152 (your ip address. If you don't include the host setting you will only be able to access from the local machine)

6. files and folders created in project

  • e2e folder: Stands for end to end. Create automated tests here.
  • node_modules folder: Stores 3rd party libraries. used only for dev. the build process only takes stuff needed from this folder
  • src folder: Contains all the source code.
  • Assets folder: Store assests such as images or icons etc.
  • Environments folder: has 2 folders one for prod and one for dev. contains environment information
  • favicon.ico is tab icon
  • index.html. is the starting point for the Angular app. the main app directive is inserted here.
  • main.ts IS the starting point of the application. the file bootstraps the main app
  • polyfill.ts used to bridge the gap between javascript and typescript
  • styles.css use to store global style settings
  • package.json Here is where the prod and dev dependencies list is kept. Can start scripts from here, etc.
  • tsconfig.json Used by the typescript compiler to build the Angular app to javascript. One key setting is target.

7. Webpack

  • Webpack is the tool that takes the source code and creates javascript bundles. files such as polyfills.bundle.js, main.bundle.js, styles.bundle.js, vendor.bundle.js, inline.bundle.js
  • Webpack does Hot Module Replacement. it only changes items that have been modified.

8. Angular version history

  • 2010, AngularJS was born a bit complicated.
  • 2016, Angular 2 came out using typescript. Not compatible with Angular JS
  • Versions went from 2.1, 2.2, 2.4 then jumped to 4. This was to align library versions. The router was on v3.3 -therefore Angular skipped 3 and went to 4.
  • Now the framework is simply referred to as simply Angular
