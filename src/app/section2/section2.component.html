13. What is Typescript

  • This section will delve into: Type annotation, Arrow functions, Interfaces, Classes, Constructors, Access modifiers, Properties, Modules
  • Typescript is a superset of javascript.
  • Uses Strong typing, brings Object-oriented features. Can catch errors at compile time. Gives us access to good tools
  • Browsers don't support typescript. typescript get transpile to javascript so the browswer understand

15. Declaring Variables

  • To declare variable you can use var or let
  • All browsers support es5 of ecmascript. es6 came out in 2015, versions then became es2016, es2017, etc.
  • let came with es6. it has different scope than a var. var is scoped to the nearest function. let is scoped to the nearest block.
  • If your typescript target is es5 let will substitute it with var.

16. Different types

  • Using Let instead of var helps Microsoft code find errors in the code we write. It will let you know if you are changing datatypes,etc
  • Type annotation will force a let to a specific variable type: number, string, etc.
  • declare: let a: number; will force the variable a to be a number (and not type any)
  • Other types: number, boolean, any, string, array (ie: number[] creates a number array), and enum

22. Constructors

  • Constructors are a way to initialize values when a new object is created. These values can be passed at creation time
  • eg: add the following to the above class after the x: and y: declaration

23. Access Modifiers

  • Sometimes we need the variables to NOT be modifiable after the object is created. To achieve this we declare the variables private
  • By default all variables are public. there are 3 types of access modifiers: private, public, and protected.
  • To do so simply add the private keyword in front of the x or y variable

24. Access Modifiers in Constructor Parameters

  • The typescript compiler has the ability to create the local variables in the constructor. This makes things look cleaner and
  • saves us from having to assign the passed values to the local variables. An access modifier is required to auto generate the local variables
  • The constructor would look like this:
  • constructor(private x?: number, private y?: number) {{leftCurly}}
