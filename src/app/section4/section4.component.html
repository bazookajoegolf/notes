40. Property Binding

  • Interpolation that we used above is a form of property binding. Below you can see examples of both interpolation and property binding
  • It is recommended that we use interpolation when you are adding text between tags and property binding when manipulating tag properties. See Example below.
  • Property binding are one way from the component to the DOM only.

41. Attribute Binding

  • For 99% of the cases attribute binding will work the same as property binding. For the odd case, such as colspan, which isn't a property
  • To solve the problem you need to prefix with attr. This targets the attribute

42. Adding Bootstrap

  • To add bootstrap to your project in install the npm package and inport it into the styles.css
  • To install npm i bootstrap --save. the --save adds bootstrap to the list of dependencies in package.json
  • in Package.json you will notice listings with ^ proceding. This indicates that you can use the latest major release. Versions are defined as major.minor.patch.
  • In styles add the line @import "~/bootstrap/dist/bootstarp.css"

43. Class Binding

  • You can add classes based on the state of your app. To do this you can use class binding
  • Inside the tag you want to modify you can add [class.active] = "isActive" where isActive is a boolean in our app.
  • you can use anything as long as it is valid.... ie: [class.mycolor] = "activeMycolor" where mycolor is a css class

44. Style Binding

  • Similar to Class binding except you user [style.color] = "isActive ? 'blue' : 'white'"

45. Event Binding

  • Event binding works the opposite direction. It goes from the DOM to the component. Some events are click, keyup, change, etc.
  • You can get access to all events. You would pass the $event object
  • See Example Below:
  • Events will bubble up the entire DOM tree unless the default behavior is blocked. This is called eventBubbling
  • To prevent, add $event.stopPropagation() in your calling method

46. Event Filtering

  • An old method to filter events would have the programmer listen for an event and if a condition of the event was met he would program something.
  • Angular has a simplified method to listen for filtered events
